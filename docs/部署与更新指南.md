# 部署与日常更新指南

本博客使用 **Hexo + Butterfly 主题**，通过 **GitHub Actions** 自动构建并部署到 **GitHub Pages**，全程零服务器、零运维。

---

## 一、首次上线部署

### 1. 推送代码到 GitHub

```bash
cd /Users/tywang/workspace/linfeiwang
git add -A
git commit -m "init: 林飞的个人博客首次提交"
git push -u origin main
```

### 2. 打开 GitHub Pages 设置

1. 打开仓库页面：https://github.com/geekvc/linfei.github.io
2. 进入 **Settings → Pages**
3. 在 **Build and deployment → Source** 里选择 **GitHub Actions**（不要选 "Deploy from a branch"）
4. 保存后，推送代码会自动触发 `.github/workflows/deploy.yml` 中定义的构建流程

### 3. 查看部署状态

- 打开仓库的 **Actions** 标签页，可以看到 "Deploy Hexo Blog to GitHub Pages" 的运行记录
- 变绿说明部署成功，几分钟后即可通过下面地址访问：
  - `https://geekvc.github.io/`（因为仓库名是 `linfei.github.io`，实际上等价于用户主页，见下方说明）
  - 或直接访问 `https://linfei.github.io/` （如果仓库名与 GitHub 用户名+`.github.io` 完全匹配时的规则，具体以 GitHub 实际生成的域名为准，登录 Pages 设置页可查看确切链接）

> 注意：由于该仓库名不是 `geekvc.github.io`（用户名.github.io），而是 `linfei.github.io`，GitHub 会将其作为**项目页面**处理。原本路径应为 `https://geekvc.github.io/linfei.github.io/`。但因为我们在 `source/CNAME` 中已经配置了自定义域名 `linfei.wang`，绑定域名解析完成后，最终会直接通过 **https://linfei.wang** 访问，不受仓库命名规则影响。域名绑定前，可以在 Pages 设置页面找到 GitHub 分配的临时访问地址用于验证。

## 二、绑定 linfei.wang 自定义域名

### 1. 腾讯云 DNS 解析配置

登录腾讯云 DNSPod / 域名解析控制台，为 `linfei.wang` 添加以下记录：

| 记录类型 | 主机记录 | 记录值 |
|---|---|---|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |
| CNAME | www | geekvc.github.io |

### 2. GitHub Pages 绑定域名

1. 仓库 **Settings → Pages → Custom domain**，填入 `linfei.wang`，保存
2. 勾选 **Enforce HTTPS**（DNS 生效后才能勾选，可能需要等待几分钟到几小时）
3. `source/CNAME` 文件已经预置好内容（`linfei.wang`），每次部署都会自动带上，无需手动在网页上重复设置

### 3. 验证

DNS 生效后（几分钟到 48 小时不等，国内解析通常较快），访问 `https://linfei.wang` 即可看到博客。

## 三、日常更新博客（本地一条命令搞定）

写完新文章后，在项目根目录执行：

```bash
cd /Users/tywang/workspace/linfeiwang
hexo new "文章标题"          # 在 source/_posts 生成新文章
# 编辑生成的 md 文件，写内容...

git add -A
git commit -m "post: 新增《文章标题》"
git push
```

`git push` 之后，GitHub Actions 会自动完成：清理 → 生成静态文件 → 部署到 Pages，通常 1-2 分钟内线上博客就会更新，全程不需要在本地运行 `hexo generate` 或手动上传任何文件。

也可以直接用内置的 npm 脚本简化：

```bash
npm run update   # 等价于 git add -A && git commit -m "update posts" && git push
```

## 五、AI 数字分身聊天机器人配置

见 [`cloudflare-worker/README.md`](../cloudflare-worker/README.md)，需要：
1. 申请 DeepSeek API Key
2. 部署一个免费的 Cloudflare Worker 作为反向代理（保护 API Key 不暴露）
3. 把 Worker 地址填入 `source/js/ai-chat-widget.js` 的 `API_ENDPOINT`
4. `git push` 重新部署即可生效

## 六、目录结构说明

```
linfeiwang/
├── _config.yml              # Hexo 站点核心配置（标题、URL、插件等）
├── _config.butterfly.yml    # Butterfly 主题个性化配置（分离配置，主题目录保持默认，方便升级）
├── source/
│   ├── _posts/               # 博客文章
│   ├── about/                 # 「关于我」页面
│   ├── resume/                 # 「个人简历」页面
│   ├── categories/, tags/    # 分类、标签页
│   ├── img/                  # 头像、封面图等
│   ├── js/ai-chat-widget.js  # AI 聊天悬浮框前端脚本
│   ├── css/ai-chat-widget.css# AI 聊天悬浮框样式
│   └── CNAME                 # 自定义域名配置
├── themes/butterfly/          # Butterfly 主题（原始文件，未做任何改动）
├── cloudflare-worker/          # DeepSeek API 反向代理 Worker 源码及部署说明
└── .github/workflows/deploy.yml # GitHub Actions 自动部署配置
```
