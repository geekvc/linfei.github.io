很多人第一反应是："你学的是英语，又不是计算机专业，为什么要自己折腾建站？"这篇文章聊聊我的动机，以及最终确定下来的技术方案，作为这个系列的开篇。

<!-- more -->

## 一、为什么不用现成的平台（知乎/公众号/小红书）

现成平台当然更省事，但对求职这件事来说，有几个明显的短板：

1. **内容所有权不在自己手里**：平台随时可能改版、限流、甚至封号，你的内容资产没有真正的掌控权
2. **无法展示"结构化能力"**：公众号推文是信息流形式，很难像独立网站一样清晰地组织"简历-文章专栏-分类标签"这种立体的个人品牌结构
3. **技术含量为零**：这一点对我来说反而是关键——英语专业学生的简历同质化程度很高，如果能在简历之外，展示一点"我愿意学习、愿意跨界折腾技术"的证据，本身就是一种差异化竞争力

## 二、我的核心需求清单

在动手之前，我先明确了几个硬性要求：

- **零成本**：学生党，不想为托管花钱
- **零运维**：不想学服务器运维、不想处理安全补丁，出了问题不知道怎么办
- **本地一条命令更新**：写完文章能快速发布，不希望流程复杂到让我"懒得更新"
- **界面要好看**：既然是给HR看的门面，视觉质感必须过关，不能是那种一眼看出是"程序员随便糊的"页面
- **能加一个AI分身**：这是最有意思的部分——让访客（尤其是HR）可以直接跟"数字分身"聊天，问一些关于我的问题，比单纯看简历更有互动性和记忆点

## 三、最终技术选型

综合以上需求，我最终确定的方案是：

| 需求 | 方案 |
|---|---|
| 博客框架 | [Hexo](https://hexo.io/)：静态博客生成器，写 Markdown 就能生成网页，不用碰数据库 |
| 主题 | [Butterfly](https://butterfly.js.org/)：卡片式设计，界面美观，社区活跃 |
| 托管 | GitHub Pages：完全免费，稳定性有 GitHub 背书 |
| 自动部署 | GitHub Actions：`git push` 后自动构建发布，不用手动上传文件 |
| 域名 | 腾讯云买的 linfei.wang，绑定到 GitHub Pages |
| AI 分身 | 智谱 AI GLM-4-Flash（永久免费）+ 腾讯云函数做安全代理 |

这几个组件全部是"零成本或近乎零成本"的方案，而且一旦搭好，日常维护只需要 `git push` 一条命令。
